# PRD：任務管理系統

## 1. 專案概述
- **背景與動機**：在快節奏的生活與工作中，人們經常需要處理多項瑣碎或重要的事務，容易遺漏或無法有效追蹤進度。本系統旨在提供一個簡單、直覺的任務管理工具，協助使用者有效追蹤與管理個人或團隊的待辦事項，提升工作與生活效率。
- **目標用戶**：需要管理個人待辦事項的一般使用者、學生或辦公室職員。
- **核心價值主張**：以輕量、直覺的操作介面，幫助使用者快速記錄與管理任務，降低記憶負擔並提升達成目標的效率。

## 2. 功能需求
系統的主要功能包含以下 5 項，並提供對應的使用者故事：

1. **新增任務**
   - 作為使用者，我希望能夠新增任務，以便記錄新的待辦事項。
2. **編輯任務**
   - 作為使用者，我希望能夠編輯已存在的任務，以便在任務細節或標題改變時進行更新。
3. **刪除任務**
   - 作為使用者，我希望能夠刪除不需要或錯誤建立的任務，以便保持任務列表的整潔。
4. **標記任務完成狀態**
   - 作為使用者，我希望能夠將任務標記為「已完成」或「未完成」，以便清楚追蹤每項任務的當前進度。
5. **檢視任務列表**
   - 作為使用者，我希望能夠檢視所有任務的列表，以便一目了然地看到所有待辦事項及其狀態。

## 3. 非功能需求
- **技術限制**：
  - 後端架構：使用 Flask 框架
  - 模板引擎：使用 Jinja2
  - 資料庫：使用 SQLite
- **效能與安全考量**：
  - 效能：頁面載入速度與操作反應需保持流暢（例如 1-2 秒內完成操作）。
  - 安全：防止常見的網頁攻擊（例如 XSS、SQL Injection），確保輸入資料的安全性與正確性。

## 4. MVP 範圍
- **Must Have（必須有）**：
  - 新增、編輯、刪除任務功能。
  - 標記任務完成狀態（已完成/未完成切換）。
  - 檢視任務列表（顯示任務標題與狀態）。
- **Should Have（應該有）**：
  - 篩選功能（顯示全部、僅顯示已完成、僅顯示未完成）。
  - 簡單的 RWD 響應式設計（支援手機版檢視）。
- **Nice to Have（可以有）**：
  - 設定任務的截止日期（Due Date）。
  - 任務分類或標籤功能。
  - 使用者註冊與登入系統（多使用者支援）。
